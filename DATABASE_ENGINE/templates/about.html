Welcome

This site is a place for exploring the past of the Oscars. With this site you can navigate through the years, the people, the movies, and the awards that have made the Oscars what they are today. This site is for all people looking to experience the history of the Oscars. It is meant to provide a simple way of navigating through the Osacars, so thank you for joining us and please enjoy.



Tools Used

In building this website we used multiple different tools to produce an effective user experience, these include: Bootstrap- Bootstrap was used to create formatting for the webpages, create a pleasent design, and implement important features such as the navigation bar. Flask MVC Architecture- This tool was used for many purposes including url routing, displaying the web pages, the webpages communication with google app engine, API calls, and database connections. Git- Git was used consistently throughout the project for team collaboration while working on the many different types of files that we created. It was effective in keeping multiple branches so that different team members could work on seperate parts of the project at the same time. Python Requests- Python Requests was used to make calls to the API and return a JSON stream containing the requested data. This is used to populate the webpage. Text Editors- Each team member used a text editor to edit and create code. These are vital in the development of this page



Results of Using Multiple Data Sources

Every database has a different structure that required parsing the data individually. An interesting result of this is that for each data model, we created unique parsing methods to display the proper data.



The Data

We ping APIs for specific information. For example for the Best Actor, Best Actress, or Best Director Award we are sending a similar API call to the WolframAlpha API which returns a list of all best actors, best actresses, or best director along with other data. Then, we parse through the JSON return values, looking for our specific data, and then send objects back to the HTML for Jinja2 to parse.
